f-react-kit

Lightweight nuclear physics for React

f-react-kit is a set of an opinionated concepts and tools for building better applications with React. It takes inspiration from Clojure’s atoms and Om’s cursors, and embraces functional programming style.

The real goal is to not propagate specific tools, but to introduce to a different and possibly better way of managing data and building things with React. This is WIP and any contribution is welcome!

A single global immutable state is a basic piece here. Immutability of the data is a safe point for applications of any scale, and functional programming style is what makes it easier to reason about the code.

How it works

The atom is a single source of truth, which holds an immutable state and exposes a basic API to manipulate the data and listen for changes.

A store provides a more specific API for writes, via dispatcher, and reads, via exposed functions, using cursors. It doesn’t store the actual data.

To reduce complexity, there’s no local state within any React components. Everytime the application state is changed, the atom will trigger an update on the root and re-render component tree. This functionality is achieved with a special higher-order component atomComponent.

In order to simplify state management across entire component tree and render efficiently there are three special higher-order components are introduced.

pureComponent consumes data passed down via props and implements shouldComponentUpdate with a shallow comparator. Incoming data is often the result of calling a read operation on the store, which does transform data.

protonComponent accepts a single cursor and is used when consuming data straight from the atom state, without intermediate transformations. The data is passed down as protonState prop. Implements shouldComponentUpdate by comparing immutable values.

nuclearComponent is basically a multi-proton component. It accepts multiple cursors and puts data into nuclearState prop. shouldComponentUpdate does comparison of each proton separately.

Libs

  • mori’s persistent data structures
  • mori includes common functional bits, but if not enough use Ramda
  • mori-transit for transforming mori’s persistent data structures into/from JSON
  • page.js is a micro client-side router
